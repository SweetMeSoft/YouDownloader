# Project README Rules

## Table of Contents
- [Required Elements](#required-elements)
- [Required Sections](#required-sections)
- [Prohibitions](#prohibitions)
- [Style](#style)

## Required Elements
- Tech badges at top. Use shields.io linking to official docs.
- Table of Contents with header anchors next.

## Required Sections
- Project Summary: Business purpose, scope.
- Functionalities: Core features list.
- Libraries/Deps: Third-party libs table + purpose.
- Core Impl: Tech description of runtime/pipelines/env.
- Versions: Target SDK, tool, API versions.
- Folder Struct: Tree diagram + brief notes.
- Design Patterns: Arch patterns (e.g., MVVM, Clean).
- Configs: Env setup + config files specs.
- Integrations: Connected APIs + external resources.

## Prohibitions
- No code snippets, class structs, impl details.
- No app status, checklists, task progress.
- No speculative features, backlog roadmap items.

## Style
- Professional technical English.
- Emoji-free.
- Standard GitHub Flavored Markdown.
